SowerBase can be configured to search for resources in specific locations through a file named 'sw_configure.sdf'. Settings in this file will override all other settings, and must exist in the application directory. This could be used to run Sower applications from a usb device, or to share resources such as modules between operating systems on the same computer. This file can be created, and edited with Sower Table Editor. Settings must exist in a table named according to the operating system on which it is to be used. These names are MSW, Linux, and MacOs. To add a setting use an identifier from the first column in the table below and set the 'Data' parameter to the desired value.

Identifier: Value: Location for: Defaults to:
Use-App-Path (Boolean) true, false Overrides all other settings, and causes Sower applications to search the application directory for all resources. Using the same directory structure presented for each item in the fourth column. User specified entries.
Data-Path Path Base location for application data files. Resources-Path + 'data'
User-Data-Path Path Bookmark, and preferences files. OS defined location.
Resources-Path Path Base location for application data files. OS defined location + 'resource'
Themes-Path Path Themes data file, and related images. Resources-Path + 'themes'
User-Keys-Path Path User keys. Resource-Path + 'keys' + 'user'
Master-Keys-Path Path Master keys. Resource-Path + 'keys' + 'master'
Publisher-Keys-Path Path Publisher keys. Resource-Path + 'keys' + 'publisher'
Module-Library-Path Path Module library.
  1. Value defined in settings.sdf.
  2. OS defined location + 'library' + 'module'
ThML-Library-Path Path ThML library.
  1. Value defined in settings.sdf.
  2. OS defined location + 'library' + 'thml'
Plugins-Path Path Plugins.
 OS defined location + 'plugins'
Application specific plugins:
Plugins-Path + 'app name'
Help-Path Path Help modules. Resources-Path + 'help'
