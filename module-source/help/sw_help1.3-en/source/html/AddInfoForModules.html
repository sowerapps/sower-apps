Sower Modules are built from ThML files (Theological Markup Language), and although ThML is specifically designed for theological texts it is suited to almost any type of book. More information on ThML can be found at www.ccel.org.

There are a few Sower specific extensions to ThML:

  1. Div1 - Div6 can have an extra attribute named 'type' which causes a specific content viewer to be used for display if a viewer of that type is available. The possible values for 'type' are:
    • Text
    • ThML
    • Html
    • Hymn
    • Lesson
    • Image
    • Map
    • Audio
    • Video
    • 2D
    • 3D

    ThML is the default, and does not need to be specified.

  2. An 'geodata' attribute can be added to the html 'img' element and should 6 variables. And should take this form:
    geodata="width in px, height in px, latitude top, latitude bottom, longitude left, longitude right"
    This will cause SowerBase to use internal geolocation data to identify places at the position of the mouse and display data associated with the place if any has been found.
  3. (img, audio, video, music, and source) elements can be used to cause media files to be stored in the module.
    <audio>
    <source src="sample-15s.wav" type="audio/wav">
    </audio>
     
  4. The Sower specific 'interface' element can be used to for Sower Binary Object files to be stored in the module using a relative path as in the 'img' element.
  5. A Sower specific element 'term.strongs' is intended for use in markup of lexicons. Used in place of the ThML 'term' element. It has 3 attributes, value (strongs number followed with a G, or H depending on the language), translit (english transliteration of the original word), origin (the word in the original language).
  6. A Sower specific element 'christ' can be used to mark Christ's words in scripture.
