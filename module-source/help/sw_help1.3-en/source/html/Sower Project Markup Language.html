Sower Project Markup Language is a markup language which specifies how a ThML file is to be created from one or more files. The table below describes the elements. All elements are processed in the order listed in the project file.

Element Description Attributes
body Represents where the body of the ThML file begins, and ends. none
div1 div1 through div6 represent a division, and depth in the text as in ThML. src - the relative path to the file that contains the content for the division.
title - the title for the division.
id - the id for the division.
div2
div3
div4
div5
div6
file A file that is to be copied to the current position of the ThML file. src - the relative path to the file.
head Represents where the head of the ThML file begins, and ends. none
info Represents where the electronic information of the ThML file begins, and ends. none
project Begins, and ends a project file. target - relative path for the ThML file that is to be created.
references Defines if scripture references should be processed, and the rules to recognize those references. Enforcing strict rules will help prevent inaccurate markup. process - true to process references, otherwise false.
cvdiv - marker that is exists between chapter, and verse numbers. Typically ':'.
span - marker that represents a span in a reference. Typically '-'.
div - marker that represents a division in a reference. Typically ','.
break - marker that represents a break in a reference. Typically ';'.
romanforchapters - true if chapter numbers are roman numerals, false otherwise.
romanforverses - true if verse numbers are roman numerals, false otherwise.
romanlowercase - true if roman numerals are only valid if lower case, otherwise false.
booknameuppercase - true if book names must always begin with a capital letter, otherwise false.
linebreakisspace - line breaks should be treated as space characters. This should be false for text files, and true for most  markup languages.
style Represents where any css style information begins, and ends. none
verbatim Text enclosed in the 'verbatim' element will be copied verbatim to the target file. none

Project file example:

<project target="book.thm">
<references process="true" cvdiv=":" span="-" div="," break=";" romanforchapters="true" romanforverses="false" romanlowercase="true" booknameuppercase="true" linebreakisspace="true"/>
<head>
<info>
<verbatim>
--Dublin Core Text--
</verbatim>
</info>
<style>
<file src="book.css"/>
</style>
</head>
<body>
<div1 src="chapter1.htm" title="Chapter 1" id="i"/>
</body>
</project>
