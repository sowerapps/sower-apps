Sower Module Writer creates modules from ThML files.

Steps to module creation:

  • Select the source file.
  • Select the destination file.
  • Enter the title of the book, and an abbreviation to be used to represent the book.
  • Enter an identifier for the book. This should be something such as a ISBN number if available. It must be unique.
  • Select the category that best fits the book.
  • Enter any group that applies to the book. Such as a particular author.
  • And if applicable enter the versification scheme used by the book, such as 'KJV'.
  • Enter the original file name of the source file in the box provided. This maybe used to follow links between books. Enter only the file name, do not include the file path.
  • Sower Modules can be encrypted. If the encryption plug-in is available the 'Encrypt' box will be enabled which can be checked and will enable a password to be entered. The module will be encrypted, and a master key will be created and stored in Sower's data folder (data-dir\keys\master). Copy commands to content viewers that are displaying any content of this module can be disabled by unchecking the box labeled 'Allow Copy'.

Click 'Build'.

Five files will be created the module (.smd), a bzipped file containing the compressed module (.bz2), a catalog card that contains information for the module so that it doesn't have to be entered again if the module is rebuilt (.card), and an html file which displays the full title of the module in that folder (Title.html). It is best for  ThML files to be stored in it's own folder. A log file containing containing the results of the build process (.log.txt).

During the build process error messages may be displayed. For most error messages the portion of the text in the source file that caused the error will follow the error code so that the problem can be located in the source file.

1. EC_UNCONTAINED_TEXT_RUN Some portion of text is not contained in an element other than the '<divN>' that it exists in. Possible cause:
<div1>text</div1>

Solution:
<div1><p>text</p></div1>
2. EC_UNCONTAINED_SCRIPTURE A scripture element does not have a mate. Possible cause:
<scripture/>verse text

Solution:
<scripture>verse text</scripture>
3. EC_UNCONTAINED_COMMENTARY A scripCom element does not have a mate. Possible cause:
<scripCom/>commentary text

Solution:
<scripCom>verse text</scripCom>
4. EC_UNABLE_TO_IDENTIFY_ID 4, - 6 all represent a situation where the scripture represented by an id could not be identified. Possible cause:
This is most likely due to an unrecognized book name, or abbreviation.

Solution:
Replace with one that is known to work.
5. EC_UNABLE_TO_IDENTIFY_PASSAGE
6. EC_UNABLE_TO_IDENTIFY_OSISID
7. EC_UNABLE_TO_LOCATE_EXTERNAL_FILE A file referenced in the source file can not be located, and has not been included in the module. Possible cause:
A missing, or misplaced file such as an image has been referenced in the source file.

Solution:
Locate the file and put it in the proper place in relation to the source file.
8. EC_UNABLE_TO_LOAD_EXTERNAL_FILE A file referenced in the source file has been located, but can not be read. Possible cause:
Another application may have the file open, and is preventing other applications from accessing the file.

Solution:
Close any applications that may be using the file.
9. EC_CORRUPT_ITEM_DATA An internal error has occurred. Possible cause:
Unknown.

Solution:
Rebuild the module.
10. EC_POSSIBLE_ENTITY_ENCODING_ERROR This is considered as a warning, as it is not certain that it is an error. Possible cause:
A '#' character has been found in a text run, and has been followed with at least one digit, and a semicolon. And is not preceded by an ampersand character. Which it must be if it is actually an encoded character.

Solution:
Insert '&' before the '#'.
